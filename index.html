WTF is API, GUI, CLI, MCP, Skills?

AI 時代大家都應該稍微瞭解的各種技術名詞，用 Uber 叫車一個例子講完。

by Tofus Wang · 2026.03

目錄

  1. API — 讓程式呼叫另一個程式
  2. GUI — 包在 API 外面的好看畫面
  3. Browser Use — 讓 AI 自己操作畫面
  4. CLI — 介於 API 和 GUI 之間
  5. MCP — 統一的工具接口標準
  6. Skills — 讓 AI 學會技能
01

API

Application Programming Interface — 讓程式可以呼叫另一個程式的接口

假設 Uber 開放了 API，你就可以寫一個 Python 程式碼，執行完就幫你叫車，不需要打開 App。

反過來說，Uber 的 App 背後其實也是透過 API 來幫你叫車的——你按畫面上的按鈕，App 的程式碼就去呼叫 API。

哪些服務故意不開放 API？
不想讓別人程式化使用的服務。比如搶票系統——如果開放 API，我就可以寫程式瘋狂搶票。
02

GUI

Graphical User Interface — 包在 API 外面，讓一般人可以點點按按的畫面

任何基於網路的服務都必須透過 API 來運作。為了讓所有人都能方便使用，就在 API 前面包一層好看的畫面——這就是 GUI。

所以搶票系統說「不提供 API」，嚴格來說是錯的，因為它自己的 GUI 一樣要透過 API 來送出訂票請求。真正的意思是：用各種手段讓你很難直接用它的 API。

03

Browser Use

當服務不提供 API，就讓程式自己去操作 GUI

十年前叫「爬蟲」，AI 時代叫 Browser Use——讓 AI 有自己使用瀏覽器的能力。不管某個服務有沒有 API，直接模擬人類操作網站。

這是 Manus、OpenClaw 🦞 這類 Agentic AI 的基本能力。但代價是：慢得要死。

🐌 Browser Use 叫車
~75 秒
開網站 5s → 驗證 10s → 登入 30s → 切頁面 5s → 輸入地址 15s → 叫車 3s
⚡ API 叫車
~3 秒
一行程式碼，幾毫秒發送 + 系統處理 2-3 秒
04

CLI

Command Line Interface — 比 GUI 快，比寫程式呼叫 API 簡單

電影裡駭客對著黑底白字畫面輸入指令？他可能只是在用 Uber 叫車。

$ uber ride request --from home --to airport

一行指令，搞定。全是國小英文單字，只是沒照文法排。

為什麼 CLI 在 AI 時代被頻繁提到？

🔒 比 GUI 穩定 — GUI 會改版、按鈕位置會變，CLI 指令不變就能跑
⚡ 比 Browser Use 快 — 一行指令 vs. 開瀏覽器登入點來點去
🧠 比 API 更適合 LLM — CLI 都是自然語言單字，LLM 不容易出錯，也更省 token

05

MCP

Model Context Protocol — 讓 AI 有一個標準化的方式使用外部工具

即使 Uber 有 API，AI 還是要先搜尋有沒有 API、翻文件、寫程式碼來呼叫。如果 Uber 推出 MCP Server，AI 就能從定義好的工具列表裡直接挑一個來用——快、準、不用猜。

聽起來很讚？但實際上有三個問題：

  • 1 被動等待 — 要等官方或社群開發 MCP Server 你才能用。沒人做？那就沒得用。
  • 2 生態碎片化 — 很多 AI 工具沒有完全支援 MCP 協議（比如不支援 OAuth），導致裝了也用不了。
  • 3 吃 Context — 所有工具的介紹、schema 都要塞進 system prompt。裝太多工具，AI 就什麼都做不了了，而且每次對話都更貴。
06

Skills

讓你的 AI「學會」技能 — MCP 三個問題的另一種解法

目前只有 Claude Code 有這個功能，但它的思路很值得看：

  • 1 不用等別人 — 任何人都能自己寫一個 markdown 檔案，放進 skills 資料夾，AI 就學會了。你甚至可以用對話的方式「教」AI 研究某個 API 或 CLI，然後讓它把學到的經驗記錄成 skill。
  • 2 不依賴生態 — 不需要 Uber 官方出 MCP Server。只要有 API 或 CLI，AI 自己研究完就能用。
  • 3 省 Context — 只把 skill 的名字和簡短描述放進 context，AI 覺得需要時才展開細節讀取。不會一開始就塞爆。
Uber 叫車 Skill 的誕生過程

你：「去研究一下 Uber 的 CLI，試試看能不能幫我叫車」
AI：（花 20 分鐘研究、踩坑、最終成功）
你：「你剛剛學到了什麼？記下來。」
AI：（寫成 skill）

下次你說「幫我叫車」→ 直接用上次成功的方法，秒完成。

📋 TL;DR

方式 速度 誰用 需要什麼
API 極快 程式 官方開放 + 寫程式碼
GUI 慢 人類 打開網站 / App
Browser Use 很慢 AI 什麼都不用，但很貴
CLI 快 人類 / AI 安裝工具 + 輸入指令
MCP 快 AI 官方或社群開發 Server
